# Async GCS Cache with Pattern Filtering for FastAPI

## Overview
Build a lightweight async TTL+LRU cache layer for a FastAPI app that reads files
from GCS. Uses `cachetools.TTLCache` as the engine, adds asyncio thread-safety,
per-key locking (thundering herd protection), and a decorator with fnmatch pattern
filtering. No per-key TTL — single TTL set at cache construction time.

## 1. cache/async_cache.py

Class: AsyncTTLCache

Constructor args:
  - maxsize: int = 256        # max number of entries (LRU evicts oldest when full)
  - ttl: int = 300            # seconds, applied uniformly to all entries

Internal state:
  - self._cache = TTLCache(maxsize=maxsize, ttl=ttl)
  - self._global_lock = asyncio.Lock()       # protects all _cache operations
  - self._key_locks: dict[str, asyncio.Lock] = {}
  - self._key_locks_lock = asyncio.Lock()    # protects the _key_locks dict itself

Methods:

  async get(key: str) -> Any | None
    - acquire _global_lock
    - return self._cache.get(key)
    - cachetools returns None automatically for expired or missing keys

  async set(key: str, value: Any) -> None
    - acquire _global_lock
    - self._cache[key] = value

  async get_key_lock(key: str) -> asyncio.Lock
    - acquire _key_locks_lock
    - create entry in _key_locks if not present
    - return the lock for that key

  def stats(self) -> dict
    - return currsize, maxsize, ttl (no lock needed — read only)

## 2. cache/decorators.py

Function: cached_gcs(cache, patterns, key_fn=None)

Args:
  - cache: AsyncTTLCache instance
  - patterns: list[str]         # fnmatch glob patterns e.g. ["*.pdf"]
  - key_fn: Callable | None     # optional custom key builder

Returns a decorator that wraps an async function whose first positional
argument is blob_path: str.

Decorator logic (in order):

  Step 1 — Pattern gate
    if not any(fnmatch.fnmatch(blob_path, p) for p in patterns):
        return await func(blob_path, *args, **kwargs)   # bypass cache entirely

  Step 2 — Build cache key
    if key_fn provided: key = key_fn(blob_path, *args, **kwargs)
    else: sha256 of f"{func.__name__}:{blob_path}:{args}:{sorted(kwargs.items())}"

  Step 3 — Fast path check (no lock)
    cached = await cache.get(key)
    if cached: return cached

  Step 4 — Acquire per-key lock
    key_lock = await cache.get_key_lock(key)
    async with key_lock:
        cached = await cache.get(key)     # double-check after acquiring
        if cached: return cached

        result = await func(blob_path, *args, **kwargs)
        if result is not None:
            await cache.set(key, result)
        return result

Use @functools.wraps(func) on the wrapper.
Log cache HIT / MISS / bypass at DEBUG / INFO level using logging.getLogger(__name__).

## 3. services/gcs.py

- Instantiate one module-level AsyncTTLCache:
    gcs_cache = AsyncTTLCache(maxsize=256, ttl=600)

- Define fetch_file(blob_path: str) -> bytes
  decorated with @cached_gcs(cache=gcs_cache, patterns=["*.pdf", "contracts/*.pdf"])
  body: use google.cloud.storage.Client to download blob as bytes
  function must be async — wrap sync GCS call with asyncio.get_event_loop().run_in_executor

## Behaviour Notes (important for correctness)

- No per-key TTL. TTL is set once at AsyncTTLCache construction. Do not add _TTLEntry
  wrappers or any manual expiry tracking — cachetools handles it entirely.

- Thread safety: cachetools.TTLCache is NOT thread-safe. Every read and write must
  be inside async with self._global_lock. No exceptions.

- Thundering herd: the per-key lock ensures only ONE coroutine fetches from GCS on
  a cache miss. All other coroutines waiting on the same key will hit the cache after
  the lock is released. This requires the double-check pattern inside the lock (shown
  in Step 4 above).

- Pattern matching uses fnmatch.fnmatch — glob syntax, case-sensitive on Linux.
  Patterns are evaluated with any(), first match wins, no priority ordering.

- The _key_locks dict grows unboundedly over time (one lock per unique key ever
  seen). This is acceptable for GCS blob paths which are finite and known.
  Do not add cleanup logic unless explicitly asked.

- GCS client should be a module-level singleton, not instantiated per request.

---

## Do Not

- Do not add a background eviction sweep — cachetools handles lazy eviction
- Do not add per-key TTL support
- Do not use redis, diskcache, or any external cache store
- Do not use threading.Lock — use asyncio.Lock throughout
- Do not use lru_cache or functools.cache — they are not async-safe
